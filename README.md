# erp-agent

A reusable agent framework for **Next.js + Tailwind + Shadcn** ERP projects.

It pairs a **global framework install** (`~/.erp-agent/`) with a **per-project
`.agent/` profile** so one workflow engine can serve many projects. The agent
runs in an IDE that supports `AGENTS.md` (e.g. Antigravity, Cursor, Claude
Code) and picks up both the project-local rules and the shared skills by
following the explicit trigger table.

- **Framework (this repo):** CLI, JSON schema, skills, templates, MCP clients.
  Lives once on your machine at `~/.erp-agent/`.
- **Profile (consumer project):** `.agent/` + `AGENTS.md` generated by
  `erp-agent init`. Contains project-specific tokens, component registry,
  patterns, conventions, and checklists.

---

## Requirements

- macOS or Linux
- `bash` 4+, `git`, `node` (any LTS) вЂ” `node` is only used by `erp-agent doctor`
  and the optional Figma/Confluence scratch scripts
- For `RV` (code review): `npx` available (used to invoke `react-doctor`)

---

## Install (one-time per machine)

```bash
git clone <this-repo-url> ~/.erp-agent
echo 'export PATH="$HOME/.erp-agent/bin:$PATH"' >> ~/.zshrc     # or ~/.bashrc
exec $SHELL
erp-agent version        # в†’ erp-agent 0.1.0
```

Update later with:

```bash
erp-agent update
```

---

## Use (per project)

```bash
cd /path/to/my-erp-project
erp-agent init           # scaffolds .agent/ + AGENTS.md (interactive)
erp-agent doctor         # validates .agent/profile.json against the schema
```

`init` asks five questions (project name, Next.js version, state library,
i18n library, shared UI package). It refuses to overwrite an existing
`.agent/` or `AGENTS.md`. After it finishes, edit `.agent/context/*.md`,
`.agent/ui/01-tokens.md`, and `.agent/ui/components/*.md` to reflect your
actual project.

For a longer walkthrough (first feature, review workflow, adding a component
to the registry, etc.) see [USAGE.md](USAGE.md).

## Triggers

The agent recognises these tokens (full list + each skill's preconditions
live in the generated `AGENTS.md`):

| Trigger | Skill | Purpose |
|---|---|---|
| `EC <figma-url> [spec=<cf-url>]` | `02-extract-context.md` | Extract Figma context (+ optional Confluence spec) |
| `PL` | `03-plan-feature.md` | Plan a feature |
| `SD` | `04-scaffold-data.md` | Scaffold the data layer (types / api / hooks) |
| `TC <figma-url> [spec=<cf-url>]` | chain 02 в†’ 03 в†’ 04 в†’ 05 в†’ 07 | Full feature: extract в†’ plan в†’ data в†’ code в†’ verify |
| `CS <confluence-url>` | `06-fetch-spec.md` | Fetch a Confluence page as Markdown |
| `VF` | `07-verify.md` | Typecheck + lint (+ tests if you ask) |
| `RV` | `08-code-review.md` | Runs `react-doctor` + project-specific manual review |
| `RF <path>` | `09-refactor-scan.md` | Report duplication/refactor scope (no code changes) |

Brainstorming is **opt-in** вЂ” the agent asks *"Brainstorm first? (yes/no)"*
before loading the brainstorm skill. TDD is **optional**.

## Hard gates

Enforced across `AGENTS.md`, `00-router.md`, and `05-transform-code.md`:

1. **No code before a plan.** If no plan exists and the user did not say
   "skip plan", load `03-plan-feature.md` first.
2. **No hardcoded tokens.** Every color / spacing / radius / typography value
   must map to an entry in `.agent/ui/01-tokens.md`.
3. **Grep before build.** Before creating a new shared component, search
   `.agent/ui/components/` for a match.
4. **Data layer before UI.** `04-scaffold-data.md` completes before
   `05-transform-code.md` starts.

## Commands

| Command | What it does |
|---|---|
| `erp-agent init` | Interactive scaffold of `.agent/` + `AGENTS.md` into the current directory. Refuses to overwrite existing files. |
| `erp-agent doctor` | Walks `.agent/profile.json` against `schema/profile.schema.json`: required fields, `type`, `minLength`, `enum`, `additionalProperties`. Exit 0 ok / 1 on any error. |
| `erp-agent update` | `git pull --ff-only` the framework. Drift detection is **not** automatic вЂ” re-run `init` into a scratch dir to diff. |
| `erp-agent version` | Prints the framework version. |
